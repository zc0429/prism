# Prism — 产品需求文档（PRD）

**版本**: V1.0
**状态**: 草稿
**最后更新**: 2026-05-06

---

## 1. 产品定位

Prism 是面向国内开发者的 **Claude Code 桌面端增强工具**。

- **核心价值**: 解决 Claude Code 在国内安装困难、配置繁琐、无法直连国产模型的痛点
- **商业模式**: 通过内置模型接入服务（充值扣费）实现盈利
- **产品形态**: Electron 桌面应用 + 内置 Web 管理面板

---

## 2. 目标用户

### 主要用户画像

国内开发者，典型特征：

- 无特殊网络环境（无法直接访问 Anthropic API）
- 不熟悉命令行配置，倾向于可视化操作
- 希望使用国产大模型替代或补充 Claude
- 对工具的开箱即用性有较高要求

### 次要用户

- 需要统一管理多个 AI 编程工具的团队
- 对 Token 消耗有预算控制需求的个人开发者

## 4. 核心功能需求

### 4.1 一键安装与配置（引流基石）

自动检测并安装前置依赖，静默完成 Claude Code 部署。

**功能点**：

- 自动检测 Node.js ≥18.x，如未安装则引导安装（调用 Node.js 官方/国内镜像分发）
- 使用国内 npm 镜像源（npmmirror.com）静默安装 Claude Code CLI
- 自动修改本地 Claude Code 配置文件，跳过 Anthropic 官方账号强制登录
- 安装进度可视化（终端输出 + 进度条双通道）
- 安装完成后引导用户进入模型配置页

**验证标准**：
- 干净 Windows/macOS 虚拟机上一键完成安装 + 配置 ≤ 3 分钟
- 安装成功率 ≥ 95%
- 安装失败时提供明确的错误信息 + 手动修复指引

### 4.2 自带密钥模式（BYOK — 免费留存）

平台封装开源工具 cc-switch 的配置管理能力，提供可视化界面管理模型和 API Key。

**功能点**：

- 内置国产模型 Base URL 模板（DeepSeek、通义千问、智谱 GLM、Moonshot 等）
- 用户填入自有 API Key，一键写入 Claude Code 配置文件
- 模型连接测试（一键检测 API Key + Base URL 是否可用）
- 模型切换：从模板列表选择或自定义 Base URL
- 配置文件可视化编辑 + 原始文本编辑双模式
- BYOK 模式下不收取任何费用

**验证标准**：
- 填入有效 Key → 模型连接测试成功
- 配置写入后重启 Claude Code 即生效
- 列出的模型模板 ≥ 6 个

### 4.3 充值扣费模式（核心盈利）

用户通过微信/支付宝充值，工具内置模型代理通道，按 Token 消耗实时扣费。

**功能点**：

- 支持微信支付 + 支付宝充值（最低充值额度 ¥10）
- 账户余额实时显示，Token 消耗明细可查
- 支持国产模型（DeepSeek、通义千问等）+ 国外主流模型（Claude、GPT-4o、Gemini 等）
- 选择模型后，工具自动将**子密钥 + Base URL** 写入本地 Claude Code 配置
- 后端调用第三方中转站 API 完成模型调度和协议转换
- 消费明细页：按时间/模型筛选，支持 CSV 导出
- 余额不足时推送通知 + 邮件提醒
- 定价透明：每个模型的 Token 单价在界面上明示

**验证标准**：
- 微信/支付宝扫码 → 支付成功 → 余额即时到账（≤ 5 秒）
- 发起模型请求 → Token 消耗实时扣减，对账无误差
- 充值流水 + 消费流水完整可查

## 5. 技术架构

### 5.1 架构原则

**不做协议转换**：Prism 不自建大模型转发服务。底层模型调度、协议适配、速率限制等全部依赖第三方中转站 API。Prism 只做凭证管理和充值结算。

### 5.2 架构图（逻辑）

```
┌─────────────────────────────────────────────┐
│  Prism Desktop (Electron)                   │
│  ┌─────────────┐  ┌──────────────────────┐  │
│  │ 安装引导    │  │ Web 管理面板          │  │
│  │ (wizard)    │  │ (React + shadcn)     │  │
│  └─────────────┘  └─────────┬────────────┘  │
│                              │                │
├──────────────────────────────┼────────────────┤
│  Prism Backend               │                │
│  ┌────────────┐  ┌───────────┴───────────┐  │
│  │ 用户系统    │  │ 配置写入服务           │  │
│  │ (注册/登录) │  │ (cc-switch + 模板)   │  │
│  └────────────┘  └───────────────────────┘  │
│  ┌────────────┐  ┌────────────────────────┐ │
│  │ 支付服务    │  │ Token 结算服务          │ │
│  │ (微信/支付)│  │ (中转站 API 封装)     │ │
│  └────────────┘  └───────────┬────────────┘ │
│                              │                │
├──────────────────────────────┼────────────────┤
│  外部依赖                     │                │
│  ┌───────────────────────────┴────────────┐  │
│  │ 第三方模型中转站 API                    │  │
│  │ (协议转换 + 模型调度 + 速率限制)        │  │
│  └────────────────────────────────────────┘  │
│  ┌────────────┐  ┌────────────────────────┐  │
│  │ 微信/支付宝│  │ npm 国内镜像           │  │
│  │ 支付网关   │  │ (npmmirror.com)      │  │
│  └────────────┘  └────────────────────────┘  │
└─────────────────────────────────────────────┘
```

### 5.3 技术栈

| 层 | 技术 | 说明 |
|------|------|------|
| 桌面壳 | Electron | 跨平台桌面应用 |
| 前端 | Next.js + React + shadcn/ui + Tailwind CSS v4 | Web 管理面板 |
| 后端 | Node.js / Go | 用户系统 + 支付 + 结算 |
| 数据库 | PostgreSQL | 用户、订单、消费记录 |
| 配置管理 | cc-switch (封装) | Claude Code 配置读写 |

## 6. 前端设计

### 6.1 设计基准

前端设计以项目根目录的 `prism-frontend.html` 为视觉和交互基准。该文件定义了：

- 色彩系统（暖灰基调 + 橙色强调）
- 侧边栏布局结构
- 按钮 / 卡片 / 导航 / 表单组件样式
- 各核心页面的布局原型

### 6.2 设计规范

| 维度 | 规范 |
|------|------|
| **基调** | 暖灰色背景 (#f5f4ef)，非暗黑终端风格 |
| **强调色** | 橙色系 (#d97757) 作为主强调色 |
| **字体** | UI: Noto Sans SC；标题: Poppins；代码: DM Mono |
| **图标** | Lucide Icons，禁止 Emoji |
| **圆角** | 6px (sm) / 10px (默认) / 14px (lg) / 18px (xl) |
| **间距** | 基于 4px 网格系统 |
| **暗色模式** | 支持，非强制 |
| **响应式** | 侧边栏 ≥768px 常驻，<768px 可折叠 |

### 6.3 核心页面

| 页面 | 路由 | 说明 |
|------|------|------|
| 品牌首页 | `/` | 产品介绍 + 下载入口 + 动态终端效果 |
| 注册 | `/sign-up` | 邮箱注册 |
| 登录 | `/sign-in` | 邮箱 + 密码登录 |
| 仪表盘 | `/dashboard` | 账户概览、余额、快速入口 |
| 工具管理 | `/dashboard/tools` | Claude Code / Cursor 等工具卡片 |
| 模型配置 | `/dashboard/models` | 模型选择 + Base URL + API Key |
| 用量明细 | `/dashboard/usage` | Token 消耗图表 + 明细表 |
| 充值 | `/dashboard/billing` | 套餐 + 微信/支付宝支付 |
| 账户设置 | `/dashboard/settings` | 个人信息、API Key 管理等 |

### 6.4 设计约束

- 色彩使用语义化 CSS 变量，不使用裸 hex
- 等宽字体仅用于命令/代码展示，UI 文本使用无衬线字体
- 所有可点击元素 ≥ 44px 触摸目标
- 焦点环在所有交互元素上可见
- 正文对比度 ≥ 4.5:1
- 不使用 0ms 过渡动画

## 7. 用户流程

### 7.1 新用户引导流程

```
下载 Prism → 一键安装 → 注册账号 → 选择模式
                                    ├─ BYOK: 输入自有 Key → 开始使用
                                    └─ 充值: 扫码充值 → 选择模型 → 开始使用
```

### 7.2 模型使用流程

```
打开 Claude Code → Prism 自动注入配置
                  → Claude Code 请求 → 中转站 → 模型返回
                  → Token 消耗记录 → 余额扣减 → 实时同步
```

## 8. 非功能需求

### 8.1 性能

- 桌面应用冷启动 ≤ 3 秒
- 页面路由切换 ≤ 300ms
- 支付结果回调 ≤ 5 秒

### 8.2 安全

- 用户 API Key 加密存储（AES-256）
- 支付通信全链路 HTTPS + 证书校验
- 不存储用户完整支付凭证（PCI DSS 最小化）
- 子密钥具备使用次数/额度上限

### 8.3 可靠性

- 支付回调采用幂等设计，防止重复扣款
- 配置写入采用事务性操作，写入失败可回滚
- 后端服务可用性 ≥ 99.5%
